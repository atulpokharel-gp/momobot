## ⚙️ Setup Instructions

### Step 1: Configure Environment Variables

```bash
# .env file in server directory

# Google Gemini API (FREE - Default)
GOOGLE_GEMINI_API_KEY=your_gemini_api_key_here

# Gmail OAuth (Optional - if using OAuth instead of API key)
GOOGLE_CLIENT_ID=your_client_id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your_client_secret
GOOGLE_OAUTH_REDIRECT_URI=http://localhost:3001/api/oauth/gmail/callback

# Alternative: OpenAI (Paid)
# OPENAI_API_KEY=sk-...

# Email Agent Settings
EMAIL_SYNC_INTERVAL=300000        # 5 minutes (default)
EMAIL_BATCH_SIZE=10               # Emails per sync (default)
EMAIL_AI_MODEL=gemini             # gemini, openai, huggingface
```

### Step 2: Get FREE Gemini API Key

1. Go to [Google AI Studio](https://makersuite.google.com/app/apikey)
2. Click "Get API Key" → "Create API Key in new project"
3. Copy the API key
4. Paste into your `.env` file as `GOOGLE_GEMINI_API_KEY`

**Free Tier Limits:**
- 60 requests per minute
- Sufficient for up to 50 connected Gmail accounts
- Perfect for testing and small deployments

### Step 3: Setup Gmail OAuth (for connecting accounts)

1. Go to [Google Cloud Console](https://console.cloud.google.com)
2. Create a new project
3. Enable Gmail API:
   - Search "Gmail API"
   - Click "Enable"
4. Create OAuth credentials:
   - Click "Credentials" (left sidebar)
   - Click "Create Credentials" → "OAuth 2.0 Client ID"
   - Select "Web application"
   - Add authorized redirect URIs:
     - `http://localhost:3001/api/oauth/gmail/callback`
     - `http://localhost:3000/oauth/gmail/callback`
     - (Your production domain)
5. Copy Client ID and Secret to `.env`

### Step 4: Database Setup

The system automatically creates tables on first run:
- `gmail_accounts` - Connected accounts
- `email_messages` - Stored emails with categorization
- `email_categories` - Custom category rules
- `email_workflow_settings` - User settings

No manual migration needed! ✅

### Step 5: Start Services

```bash
# Terminal 1: Start backend (with email monitoring)
cd server
npm start

# Terminal 2: Start client
cd ../client
npm start

# Terminal 3: Start agent (for real-time monitoring)
cd ../momobot-agent
npm start
```

## 🚀 Usage Guide

### For Users

#### 1. Connect Gmail Account
1. Go to Email Workflow → Gmail Accounts
2. Click "+ Connect Gmail Account"
3. Login with your Google account
4. Grant permission to access emails
5. Account appears in the list

#### 2. View Emails
1. Go to Email Workflow → Inbox
2. **Left sidebar** shows:
   - Category breakdown
   - Priority filters
   - Statistics
3. **Center panel** shows email list:
   - Sender name and email
   - Subject
   - AI summary (generated by Gemini)
   - Category badge (Work, Personal, etc.)
   - Priority indicator (High/Normal/Low)

#### 3. Read & Reply
1. Click an email to open details
2. See full message, metadata, and AI summary
3. If Important: View suggested reply (AI-generated)
4. Copy reply or approve to send

#### 4. Configure Settings
1. Go to Email Workflow → Settings
2. **AI Model**: Choose from (Gemini recommended - free)
3. **Auto-Categorize**: Enable to auto-sort emails
4. **Auto-Reply**: Disabled by default (safety)
5. **Sync Interval**: How often to check (default: 5 mins)

## 🔒 Security Best Practices

### 1. OAuth Never Shared
- User login via Google OAuth
- We never see passwords
- Tokens stored securely in DB
- RefreshToken for long-term access

### 2. Email Privacy
- Emails stored encrypted in database
- API communications via HTTPS
- Rate limiting prevents brute force
- User-scoped queries (can't see others' emails)

### 3. API Key Protection
- Never commit `.env` file
- Use environment variables
- Rotate keys monthly
- Monitor API quota usage

### 4. Auto-Reply Safety
- Disabled by default
- Must manually approve replies
- Review log before enabling
- Can set auto-reply templates

## 📈 API Endpoints Reference

### Gmail Accounts
```bash
# Get connected accounts
GET /api/email-workflow/accounts
Authorization: Bearer <token>

# Connect new account
POST /api/email-workflow/accounts/connect
{
  "email": "user@gmail.com",
  "refreshToken": "...",
  "accessToken": "..."
}

# Set as default
POST /api/email-workflow/accounts/{accountId}/default

# Remove account
DELETE /api/email-workflow/accounts/{accountId}
```

### Email Messages
```bash
# Get emails (with filtering)
GET /api/email-workflow/emails?category=work&priority=high

# Get single email
GET /api/email-workflow/emails/{emailId}

# Store email
POST /api/email-workflow/emails
{
  "gmailId": "...",
  "fromEmail": "...",
  "subject": "...",
  "body": "...",
  "autoProcess": true
}

# Mark as replied
PATCH /api/email-workflow/emails/{emailId}/replied
{ "reply": "..." }
```

### Settings
```bash
# Get settings and available models
GET /api/email-workflow/settings

# Update settings
PUT /api/email-workflow/settings
{
  "aiModel": "gemini",
  "autoCategorize": true,
  "autoReplyImportant": false,
  "syncInterval": 300
}
```

## 🐛 Troubleshooting

### Issue: "API key not configured"
**Solution:**
```bash
# Check .env has GOOGLE_GEMINI_API_KEY
grep GOOGLE_GEMINI_API_KEY .env

# If missing:
# 1. Get key from https://makersuite.google.com/app/apikey
# 2. Add to .env
# 3. Restart server
```

### Issue: "Gmail account won't connect"
**Solution:**
1. Ensure Gmail OAuth credentials are set in `.env`
2. Check redirect URI matches exactly
3. Verify Google Cloud project has Gmail API enabled
4. Try logging out of Google and back in

### Issue: "Emails not syncing"
**Solution:**
1. Check agent is running: Process shows `emailMonitoring` task
2. Verify account has `sync_enabled = 1`:
   ```bash
   SELECT email, sync_enabled FROM gmail_accounts;
   ```
3. Check API quota isn't exceeded
4. Look at server logs for errors

### Issue: "High API usage costs"
**Solution:**
- Use **Gemini** (free) instead of OpenAI
- Increase `syncInterval` (check emails less frequently)
- Reduce `batchSize` (load fewer emails per sync)
- Disable auto-categorization if not needed

### Issue: "Emails not categorizing correctly"
**Solution:**
1. Review the AI summary for understanding
2. Create custom category rules
3. Switch to OpenAI GPT-4 for better accuracy (paid)
4. Provide training examples in settings

## 📱 WebSocket Events

Real-time updates via Socket.IO:

```javascript
// Client listeners
socket.on('email:newEmail', (email) => {
  // New email arrived
});

socket.on('email:emailCategorized', (data) => {
  // Email has been categorized
  console.log(data.category, data.priority);
});

socket.on('email:syncComplete', (stats) => {
  // Sync cycle finished
  console.log(`Synced ${stats.count} emails`);
});

// Client emitters
socket.emit('email:startMonitoring', {
  userId: 'user123',
  token: 'auth_token'
});

socket.emit('email:stopMonitoring', {
  userId: 'user123'
});
```
